What is Keeper University?

The first consumer-facing academic finance platform built for students and staff

Academic institutions worldwide are beginning to accept cryptocurrency, but the operational path is still messy. Students hold assets across wallets, protocols, and exchanges, while universities need to split incoming funds into multiple internal channels. Keeper University presents that complexity as a consumer-friendly experience on top of the KeeperKit SDK.

Our AI agent builds a workflow that harvests funds from every source, swaps them into the university's required token, bridges across chains if needed, and deposits exactly the fee amount into the university endowment wallet. How it works

From scattered crypto to paid tuition — in four steps

1

Describe your intent

Tell the assistant exactly what needs to happen. A student can explain where the money lives, a bursar can explain how the campus wants to receive it, and the workflow builder can turn that into a usable plan.

2

Agent builds a workflow graph

Using the KeeperKit SDK, the agent constructs a directed graph of trigger nodes, swap nodes, condition nodes, and destination nodes. It validates the graph locally before submission so orphan nodes, dangling edges, and missing triggers are caught early.

3

Execute on KeeperHub

The workflow is submitted to KeeperHub via client.executeWorkflow(id). An execution ID is returned immediately, and the platform can poll status with getStatus() and getLogs() until the run is complete.

4

Funds arrive, receipts generated

The university receives the exact fee amount in its preferred token. A shareable workflow link and a readable execution log are generated for both the student and the bursar's office.

Built on KeeperKit SDK

The same SDK we ship to developers powers this platform

KeeperKit is the official TypeScript SDK for KeeperHub — a typed, ergonomic interface for workflows, executions, chains, integrations, marketplace listings, and direct execution. 9 core modules, 2 client styles, and a retrying HTTP client with typed error models.

Workflows

Full lifecycle management

List, create, update, delete, enable, disable, execute, duplicate, download, and go-live. Graph builders with createTriggerNode, createActionNode, and createConditionNode.

Executions

Status polling & logs

Poll execution status, retrieve structured logs, and use waitForExecution to block until terminal state. Automatic retry on 404 during indexing windows.

Auth

Pluggable providers

ApiKeyAuth, OAuthBearerAuth, and SessionAuth providers. Keys start with kh_ and are validated at initialization.

Transport

Retrying HTTP client

Native fetch with auth headers, query serialization, and typed error mapping. GET retries by default, writes only with explicit retryWrites.

Getting Started

How to sign up for KeeperHub and generate your API key

To use the AI Workflow Builder, you need a KeeperHub account and an API key. The key is used to authenticate every SDK call against the KeeperHub protocol. Here's how to set everything up in under 5 minutes.

01

Create a KeeperHub Account

Visit app.keeperhub.com and sign up with your university email address. KeeperHub supports SSO for partnered institutions — check if your university is on the list for one-click enrollment.

02

Navigate to Developer Settings

Once logged in, click your profile avatar in the top-right corner, then select "Developer Portal" from the dropdown. This opens the API management panel where you can create and manage your organization's API keys.

03

Generate an API Key

Click "Create New Key", give it a descriptive name like "University Fee Agent", and select the appropriate scopes (workflows:read, workflows:write, executions:read). Your key will be displayed once — copy it immediately. All KeeperHub API keys start with kh_.

04

Use the Key in Keeper University

Paste your API key into the Keeper University settings panel, or set the KEEPERHUB_API_KEY environment variable. The platform validates the key on first use and connects your account to the AI workflow builder. You're ready to create your first workflow.
